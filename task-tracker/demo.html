Standalone browser demo. Tasks live in a JavaScript array, so they reset on refresh. The full project swaps this for REST calls to a Flask server backed by SQLite — see the source and API write-up.

How it worksBehind the demo

01

REST API

Flask exposes the task list as CRUD endpoints — GET, POST, PUT, DELETE — so the front end never talks to the database directly.

02

Parameterized SQL

Every query binds its values as parameters instead of building strings, so a task title can't change the shape of the statement it lands in.

03

Validated input

Titles are required and length-capped, and priority has to be one of three known values — checked on the server, not just in the browser.
