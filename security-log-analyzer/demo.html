Security Log Analyzer

A SIEM-style detection engine that turns raw authentication logs into ranked, MITRE ATT&CK-tagged alerts — the core of a SOC analyst's day. This page runs the real detection rules in your browser over a sample day of … synthetic log events. Adjust the detection threshold and watch the alerts recompute, then click an alert to trace it back to the raw events.

How it works: the same rules run here and in the Python CLI (analyze.py) — brute force and its success (compromise), password spraying, port scanning, impossible travel (geo-velocity), and off-hours privileged access — each mapped to a MITRE ATT&CK technique. Logs are synthetic; no real data is involved. See the source and detection write-up.

How it worksBehind the demo
01

Sliding-window rules

Brute force, spraying, and port scans are counted over a moving time window. Every threshold lives in one DEFAULTS block, because tuning sensitivity is the actual job.

02

Geo-velocity

Two logins for one user become a haversine distance over elapsed time. Anything faster than a plane is flagged as impossible travel.

03

Ranked and mapped

Each alert carries a severity and a MITRE ATT&CK technique ID, so a genuine compromise sorts above the noise and speaks the language every SOC uses.
